# 09 分阶段练习与参考答案

这组练习按“纸面推导 -> 普通 Python -> 阅读输出 -> 安全扩展”递进。先独立完成，再看每节后的参考答案。

## 1. 入门：认清对象边界

### 练习 1

说明 Stage、Prim、API Schema、Articulation、RenderProduct、Annotator、Writer 各自是什么，并给出本项目中的一个例子。

### 练习 2

为什么 `SimulationApp` 必须在 `omni.replicator` 等模块前启动，而 profile/timing 模块可以提前加载？

### 参考答案

1. Stage 是组合场景；Prim 是场景节点；API Schema 给 Prim 增加行为/属性；Articulation 是关节刚体系统；RenderProduct 是指定相机和分辨率的渲染输出；Annotator 读取 RGB/语义等数据；Writer 组织并保存标注。例子分别可用默认 overlay、Camera Prim、SemanticsLabelsAPI、四关节挖掘机、SemanticCapture、semantic_segmentation、SemanticDatasetWriter。
2. `omni`/Replicator 依赖已初始化的 Kit 运行时；profile/timing 只用标准库和普通数据结构。把后者前置能在启动 GPU 前发现参数错误，也让单元测试不依赖 Isaac。

## 2. 时间数学

### 练习 3

给定 `physics_hz=60`、`capture_fps=15`、捕获初始帧，共 6 帧。列出每帧的 data step 与 dataset time。

### 练习 4

同样参数，但关闭初始帧。第 0 帧和第 5 帧分别是什么时间？

### 练习 5

为什么 `physics_hz=100`、`capture_fps=30` 被拒绝？若放宽为舍入步数，会产生什么风险？

### 参考答案

3. 每次间隔 4 步。data step 为 0、4、8、12、16、20；时间为 0、0.0666667、0.1333333、0.2、0.2666667、0.3333333 秒。
4. 整体后移一个间隔。第 0 帧为 step 4、约 0.0666667 秒；第 5 帧为 step 24、0.4 秒。
5. 100 不能整除 30。若每帧舍入，步距会在 3/4 间变化，简单实现还会积累相位误差；帧号到物理步不再是当前的精确线性整数映射。

## 3. 轨迹插值

### 练习 6

默认轨迹 t=0 到 t=1.25 的 `cab` 从 -2.4° 到 17.6°。计算 t=0.25 秒的目标。

### 练习 7

若轨迹时长 5 秒，`hold` 与 `loop` 在 t=6.25 秒分别采样哪个轨迹时刻？

### 练习 8

为什么 loop 要求首尾位置相同？默认轨迹能否 loop？

### 参考答案

6. \(\alpha=0.25/1.25=0.2\)，目标为 \(-2.4+0.2\times20=1.6°\)。
7. hold 为 5.0 秒；loop 为 1.25 秒。
8. 不闭合时周期边界会产生位置跳变，直接位置命令和图像都会不连续。默认轨迹 t=0 与 t=5 四个值相同，可以 loop。

## 9. 二次开发挑战
