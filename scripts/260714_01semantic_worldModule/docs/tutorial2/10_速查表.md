## 3. 时间公式

```text
steps_per_capture = physics_hz / capture_fps
dataset_step       = physics_step - dataset_origin_step
dataset_time       = dataset_step / physics_hz
```

初始帧开启：

```text
data_step(frame_id) = frame_id * steps_per_capture
```

初始帧关闭：

```text
data_step(frame_id) = (frame_id + 1) * steps_per_capture
```

静态模式：

```text
data_step(frame_id) = 0
```

## 4. 每步与每帧顺序

```text
每个物理步：
  world.update(next_dataset_time)
  motion.before_physics_step(next_dataset_time)
  SimulationApp.update()
  motion.after_physics_step(dataset_time)

每个输出帧：
  推进到目标 data step
  冻结 Timeline
  读取相机与 motion
  构造 CaptureContext
  arm Writer Ledger
  Replicator step(delta_time=0)
  等待完成并取得 Receipt
  断言世界仍冻结
  写 motion_state.jsonl
```

## 5. 状态机

```text
World:

Camera:

        -> WRITER_ATTACHED -> CAPTURING -> CLOSED
```

## 11. 修改影响面

| 修改 | 至少同步检查 |
|---|---|
| mapping 类别 | Stage 标签、mapping、Writer、validator、训练配置 |
| 时间策略 | CaptureTiming、WorldScheduler、manifest、validator、tests |
| 关节数量/顺序 | profile、CSV、stage validator、motion、output validator、tests |
| renderer/profile | launch keys、Carb settings、required read-back、sampling validator |
| 输出 schema | Writer、manifest version、validator、tests、消费者 |
| 多相机/异步 | Context identity、Ledger、barrier、文件名、manifest、validator |

## 12. 三条不要忘记的原则

1. **先回读物理接受的状态，再构造帧上下文。**
2. **采集期间 Timeline 必须保持冻结。**
3. **运行完成不等于数据合格，独立 validator PASS 才算交付。**
