# 10 项目速查表

## 1. 常用命令

本地测试：

```powershell
python -m unittest discover -s tests -v
```

远端采集：

```bash
./run_capture_remote.sh \
  --config configs/capture_motion_camera02.json
```

验证输出：

```bash
/root/isaacsim/python.sh validate_semantic_output.py \
  --output output/semantic_capture_camera02 \
  --mapping configs/semantic_mapping_Sim_Fangshan_07_native.json \
  --expected-frames 50
```

## 2. 业务配置规则

```text
唯一命令行选项       --config <json>
配置 schema          1
JSON 字段命名         snake_case
相对路径基准          主配置文件所在目录
未知字段              拒绝
缺少字段              拒绝
命令行覆盖            不允许
额外 Kit 参数         不允许
```

完整样本：

```text
configs/capture_motion_camera02.json
```

## 3. 核心字段

```text
usd                    Stage 文件
mapping                稳定语义映射
camera_prim_path       Camera 绝对 Prim 路径
renderer               RealTimePathTracing 或 PathTracing
render_profile         版本化渲染配置
output                 输出目录
frames                 输出帧数
width / height         图像分辨率
physics_hz             固定物理频率
capture_fps            数据采集频率
capture_mode           static 或 motion
enable_motion          关节控制开关
trajectory             四关节轨迹 CSV
joint_profile          Articulation 控制契约
headless               是否无窗口运行
strict_stage           Stage 严格预检
strict_mapping         语义严格映射
```

`capture_mode` 与 `enable_motion` 不强制对应。只有：

```text
capture_mode == motion and enable_motion == true
```

才会创建运动控制器。

## 4. 时间公式

```text
steps_per_capture = physics_hz / capture_fps
dataset_step       = physics_step - dataset_origin_step
dataset_time       = dataset_step / physics_hz
```

初始帧开启：

```text
data_step(frame_id) = frame_id * steps_per_capture
```

初始帧关闭：

```text
data_step(frame_id) = (frame_id + 1) * steps_per_capture
```

静态模式：

```text
data_step(frame_id) = 0
```

## 5. Camera 规则

```text
必须显式提供 camera_prim_path
必须是绝对 USD Prim path
Prim 必须存在且为 UsdGeom.Camera
可以位于任意父级
不要求位于 cab 下
不要求在 motion 模式中移动
```
