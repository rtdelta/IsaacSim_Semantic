# 06 配置与运行实战

## 1. 唯一启动方式

采集程序只接受一个业务选项：

```bash
./run_capture_remote.sh \
  --config configs/capture_motion_camera02.json
```

不再允许把 `frames`、`camera_prim_path`、`output` 等参数追加到命令行。需要改变任务时，复制一份配置文件并修改副本。

例如创建三帧 smoke test 配置：

```bash
cp configs/capture_motion_camera02.json configs/capture_smoke_camera02.json
```

然后编辑 `capture_smoke_camera02.json`：

```json
{
  "frames": 3,
  "width": 640,
  "height": 360,
  "output": "../output/tutorial_smoke_camera02"
}
```

上面只展示需要关注的字段；实际 JSON 必须保留样本中的全部字段。

## 2. 完整业务配置

项目提供可复制的完整样本：

```text
configs/capture_motion_camera02.json
```

它包含：

- Stage 与语义映射；
- 任意层级 Camera 的绝对 Prim 路径；
- renderer 与 render profile；
- 输出目录、帧数和分辨率；
- 物理频率、采集频率和初始帧策略；
- 运动开关、轨迹和关节 profile；
- headless、严格预检和运行时 ID 设置。

JSON 字段使用下划线命名，例如：

```json
{
  "camera_prim_path": "/root/Xform/operator_cab_mesh/Camera_02",
  "capture_initial_frame": true,
  "strict_stage": true
}
```

字段缺失、额外字段、错误类型和不支持的枚举值都会在 Isaac Sim 启动前报错。程序不会静默忽略拼写错误。

## 3. 路径解析

配置文件内部的相对路径以配置文件所在目录为基准。

配置位于 `configs/` 时：

```json
{
  "usd": "Sim_Fangshan_07_capture_overlay.usda",
  "trajectory": "../trajectories/excavator_motion_01.csv",
  "output": "../output/semantic_capture_camera02"
}
```

分别解析为：

```text
项目/configs/Sim_Fangshan_07_capture_overlay.usda
项目/trajectories/excavator_motion_01.csv
项目/output/semantic_capture_camera02
```

`run_capture_remote.sh` 会在切换到隔离的 `.runtime/home` 之前把主配置路径转成绝对路径，因此不会受运行时工作目录影响。

## 4. Camera 配置

Camera 必须通过 `camera_prim_path` 明确指定：

```json
{
  "camera_prim_path": "/root/Camera_03"
}
```

要求：

- 使用绝对 Prim 路径；
- Prim 已存在；
- Prim 类型是 `UsdGeom.Camera`。

Camera 可以位于 Stage 任意父级下。父子关系只决定 Camera 的实际世界位姿是否继承父级运动，不决定它能否采集。

## 5. 运动组合

`capture_mode` 和 `enable_motion` 保持独立，不强制一一对应。现有执行逻辑为：

```text
motion_enabled = capture_mode == "motion" and enable_motion
```

因此只有二者分别为 `"motion"` 和 `true` 时才创建关节运动控制器。其他组合不会在配置加载阶段被禁止。

常规运动任务：

```json
{
  "capture_mode": "motion",
  "enable_motion": true,
  "trajectory_mode": "hold"
}
```

静态重复采集：

```json
{
  "capture_mode": "static",
  "enable_motion": false
}
```

## 6. 渲染配置

实时路径追踪：

```json
{
  "renderer": "RealTimePathTracing",
  "render_profile": "render_realtime_pathtracing_720p.json",
  "warmup_render_frames": null,
  "rt_subframes": null
}
```

高质量 Path Tracing：

```json
{
  "renderer": "PathTracing",
  "render_profile": "render_pathtracing_720p_64spp.json",
  "warmup_render_frames": null,
  "rt_subframes": null
}
```

`null` 表示沿用 render profile 的值；整数表示覆盖。配置中的 `renderer` 必须与 profile 内声明的 renderer 一致。

## 7. 输出目录安全

默认建议：

```json
{
  "output": "../output/20260723_motion_camera02_run001",
  "overwrite": false
}
```

当 `overwrite` 为 `false` 时，非空输出目录会阻断任务。设为 `true` 只允许继续写入，不会清理旧帧；旧文件可能导致最终验证失败。因此生产任务应优先使用新的输出目录。

## 8. 运行清单

输出目录中的 `run_config.json` 使用 schema v4，记录：

- 输入业务配置的绝对路径、大小和 SHA-256；
- 路径解析及 render profile 覆盖后的 `effective_config`；
- Stage、映射、轨迹和关节 profile 的文件记录；
- Stage/Articulation 预检；
- 实际生效的渲染设置；
- Camera、运动、Writer 和完成状态。

这使一次运行不依赖后来可能被修改的原始配置内容。

## 9. 验证

采集完成后仍使用独立验证器：

```bash
/root/isaacsim/python.sh validate_semantic_output.py \
  --output output/tutorial_smoke_camera02 \
  --mapping configs/semantic_mapping_Sim_Fangshan_07_native.json \
  --expected-frames 3
```

验证器是数据集验收工具，不属于采集程序的业务启动参数，因此仍保留自己的命令行选项。
